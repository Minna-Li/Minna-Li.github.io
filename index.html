Python编程-Numpy库

Numpy库的层级结构

记住Numpy的设计原则是

  1. 基础、高频功能 → 提升到顶级（np.dot, np.array）
  2. 专业、独立功能 → 作为子包导出（np.linalg, np.fft）
    • linalg : 线性代数，因为用户需要明确知道在使用线性代数
    • fft : 傅里叶变换，专业功能
    • random : 随机数生成，明确的功能域
    • polynomial : 多项式
    • ma : 掩码数组，高级功能
  3. 实现细节 → 完全隐藏（core, compat）

Numpy广播机制

numpy广播对象必须是numpy数组，如果是列表或者元组需要先转为数组！

  • case 1 : 纯列表的加法和乘法是元素拼接和重复！(元组情形类似)

广播的本质：数组自动扩展维度并逐元素运算，但所有广播行为必须遵从两条核心规则：

  1. 规则1：维度对齐(从右到左匹配维度)
    将两个数组从最右侧开始比对，满足以下条件之一即可广播：
    (1). 维度大小相等
    比如数组A形状 (2,3),数组B形状 (3,)，最右侧维度大小都为3，可以广播
    比如数组A形状 (2,3),数组B形状 (2,)，最右侧维度大小A为3，B为2(不同且非1)，无法广播
    (2). 其中一个数组的维度大小为1
  1. 规则2： 维度扩展(补1原则)
    若两个数组的维度数不同，先给维度数少的数组往左侧补 1，直到维度数一致，再应用规则1

Python编程-Python层级概念

一、Python生态（物理层级）

基本概念：

  • 模块module ：一个.py文件就是一个模块
  • 包package ： 由多个相关的模块组成的目录就是包，包下必须包含__init__.py文件
  • 库library : 多个功能相关的包封装构成一个库，可以直接上传到PyPI（Python Package Index），这是 Python 官方的包仓库
  • 框架framework：提供一个架构，遵循特定的规则写代码

包设计原则:

  1. 扁平优于嵌套（但不要太扁平）
  2. 每个包/模块有单一明确的责任
  3. 在顶层的__init__.py 文件中要做以下事情：
    • 控制公开的API
    • 使用__all__明确导出内容
    • 定义延迟加载提升性能，比如numpy中的__getattr__()
    • 提供清晰的导入路径
